codonyat · viral NGS variant calling

codon-yat

A codon-aware amino acid variant typer from SAM alignments of viral NGS data. Standalone Python package and Nextflow pipeline for end-to-end amplicon variant analysis.

GitHub repository

Highlights

  • CIGAR-aware codon extraction that correctly handles insertions, deletions, and soft clips.
  • Strand-ratio balancing and Shannon entropy tracking at every codon position.
  • TSV/XML output compatible with legacy Perl diagnostics, plus Python objects for downstream tooling.
  • Validates all inputs before parsing — malformed SAM/FASTA/amplicon data fails fast.
  • Configurable target protein via --protein (defaults to RT).
  • Complete Nextflow DSL2 pipeline with QC, trimming, decontamination, alignment, and optional dedup.

Nextflow pipeline

The repository includes a production-ready Nextflow DSL2 pipeline wrapping codonyat with upstream processing.

FASTQ → FastQC → fastp → FastQC → bowtie2 → samtools sort → [Picard dedup] → codonyat → MultiQC

Quick start

# Build the codonyat container
docker build -t codonyat:latest .

# Run with your data
nextflow run . -profile docker \
    --input samplesheet.csv \
    --reference data/HXB2R.fasta \
    --amplicons data/amplicons.tsv

# Run with bundled test data
nextflow run . -profile docker,test

Samplesheet format

A CSV file with columns sample_id, fastq_1, and fastq_2 (leave fastq_2 empty for single-end):

sample_id,fastq_1,fastq_2
sample1,/path/to/sample1_R1.fastq.gz,/path/to/sample1_R2.fastq.gz
sample2,/path/to/sample2_R1.fastq.gz,

CLI usage

Install the package and run the variant caller directly on a SAM file:

pip install .

codonyat sample.sam reference.fasta amplicons.tsv \
    --protein RT \
    --ratio-upper 3.16 \
    --entropy-threshold 0.5

Output: sample.sam.tsv (columns: FILE, REFERENCE, PROTEIN, VARIANT, POSITION, FREQ, FWCOV, RVCOV, TOTALCOV, RATIO) and sample.sam.xml with per-position depth and variant elements.

The lightweight codonyat-runner entry point supports --csv-path / --xml-path overrides for custom output locations:

codonyat-runner sample.sam reference.fasta amplicons.tsv \
    --protein RT --ratio-upper 3.1 --csv-path results.tsv

Reference FASTA format

The FASTA header must include protein annotations in Name(Description):start-end format, separated by semicolons:

>K03455|HIVHXB2CG PR(Protease):2253-2549;RT(Reverse Transcriptase):2550-3869;INT(Integrase):4230-5093
TGGAAGGGCTAATTCACTCCCAACGAAGAC...

Coordinates are 1-based positions on the reference sequence. The --protein flag selects which annotation to use for variant calling.

Package API

Import the core objects for tighter integration:

from aa_caller import SamContainer, FullReference, parse_amplicons

Use call_variants to run the full pipeline from Python:

from pathlib import Path
from aa_caller import call_variants

result = call_variants(
    sam_path=Path("reads.sam"),
    reference_path=Path("reference.fasta"),
    amplicons_path=Path("amps.tsv"),
    protein="RT",
    ratio_upper=3.2,
)

print(result.csv_path, result.xml_path)
print(result.container.variants.keys())

Adapt an argparse.Namespace or mapping with call_variants_from_args:

from argparse import Namespace
from aa_caller import call_variants_from_args

args = Namespace(
    sam_file="reads.sam",
    reference_file="reference.fasta",
    amplicons_file="amps.tsv",
    protein="RT",
    ratio_upper=3.2,
)

call_variants_from_args(args)
