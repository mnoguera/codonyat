aa-caller · RNA-temporal variant tracking

Modern RT diagnostics that stay faithful to the Perl original

`aa-caller` reproduces the `stand_alone_fullkkwr2` workflow while giving you importable Python primitives (`SamContainer`, `FullReference`, `Amplicon`, etc.), extra validation, and exports that downstream tools can consume consistently.

TSV / XML

Identical outputs to the legacy Perl script so your downstream pipeline keeps running.

Strand balance

Tracks forward/reverse counts, ratios, and entropy per position for richer diagnostics.

CLI + API

Use the `aa-caller` command-line or import its helpers directly inside automated workflows or notebooks.

Installation

Install locally before running the CLI or importing the package:

pip install ./aa_caller

or publish via PyPI/Twine and install from the registry.

The CLI entry point then becomes available as aa-caller.

CLI usage

Invoke the tool with a SAM file, a FASTA reference, and an amplicon TSV:

aa-caller sample.sam reference.fasta amplicons.tsv

Optionally tune the strand-ratio window and entropy threshold:

aa-caller sample.sam reference.fasta amplicons.tsv --ratio-upper 3.16 --ratio-lower 0.32 --entropy-threshold 0.5

The CLI writes sample.sam.tsv and sample.sam.xml with the same columns and structure as the original script.

Package API

You can skip the CLI and import the core objects for tighter integration:

from aa_caller import SamContainer, FullReference, parse_amplicons

This lets you run the core logic inside Python jobs, unit tests, or notebooks without shelling out.

Development & CI parity

Install the project with the dev tooling so local runs match CI:

pip install --upgrade pip
pip install -e .[dev]

That gives you pytest and ruff, which the workflow runs on every push/PR:

ruff check .
python -m pytest

If you ever need to understand the workflow, see the CI definition.

Testing

  • Validation helpers cover codon translation, input validation, and simple parser paths.
  • The pytest suite lives in tests/test_app.py and aims to keep the legacy behavior locked down.
  • Run python -m pytest locally, preferably after ruff check ..
