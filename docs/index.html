codonyat · RNA-temporal variant tracking

RT-aware variant detection built for automation

`codonyat` drives the RT variant calling pipeline: it validates the SAM/reference/amplicon inputs, ingests every mapped read into a `SamContainer`, and calculates strand-aware ratios, entropy, and TSV/XML exports that downstream workflows consume without extra parsing.

Consistent outputs

Generates TSV and XML diagnostics with the same schema the downstream tools expect so you can compare runs reliably.

Balance metrics

Tracks forward/reverse depth per codon, records variants per amplicon, and flags balanced states via configurable ratio thresholds.

CLI + API

Run `codonyat` from the shell or import `SamContainer`, `FullReference`, and `parse_amplicons` in Python for tighter automation.

Installation

Install locally before running the CLI or importing the package:

pip install .

or publish via PyPI/Twine and install from the registry.

The CLI entry point then becomes available as codonyat.

CLI usage

Invoke the tool with a SAM file, a FASTA reference, and an amplicon TSV:

codonyat sample.sam reference.fasta amplicons.tsv

Optionally tune the strand-ratio window and entropy threshold:

codonyat sample.sam reference.fasta amplicons.tsv --ratio-upper 3.16 --ratio-lower 0.32 --entropy-threshold 0.5

The CLI writes sample.sam.tsv and sample.sam.xml with columns for FILE, REFERENCE, PROTEIN, VARIANT, POSITION, FREQ, FWCOV, RVCOV, TOTALCOV, and RATIO so every downstream consumer sees the same diagnostics each time.

Functionality

At its core the tool:

  • Validates the input files (SAM, FASTA, amplicon TSV) to fail fast on malformed datasets.
  • Loads every mapped read, extracts codons per position, and aggregates counts per strand and amplicon.
  • Computes per-position strand ratios, entropy, and amino-acid summaries, flagging which variants are within the configured balance window.
  • Emits TSV and XML reports with depth, ratio, and per-variant details so you can load the outputs into downstream parsers or visualization tools immediately.

That workflow keeps the diagnostics stable while giving you programmatic objects for deeper orchestration.

Package API

You can skip the CLI and import the core objects for tighter integration:

from aa_caller import SamContainer, FullReference, parse_amplicons

This lets you run the core logic inside Python jobs, unit tests, or notebooks without shelling out.

Development & CI parity

Install the project with the dev tooling so local runs match CI:

pip install --upgrade pip
pip install -e .[dev]

That gives you pytest and ruff, which the workflow runs on every push/PR:

ruff check .
python -m pytest

If you ever need to understand the workflow, see the CI definition.

Testing

  • Validation helpers cover codon translation, input validation, and simple parser paths.
  • The pytest suite lives in tests/test_app.py and aims to keep the legacy behavior locked down.
  • Run python -m pytest locally, preferably after ruff check ..
